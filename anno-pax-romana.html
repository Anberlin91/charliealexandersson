Overview

Redesigning complex strategy game interfaces

Anno Pax Romana is a strategy game with deeply layered systems — resource management, city building, trade routes, and diplomacy. This case study explored how the existing UI could be redesigned to improve information hierarchy, reduce cognitive load, and make complex data more accessible to players.

The project was self-initiated as a design exercise to apply systems design thinking to an existing game's interface challenges.

Problem

Information overload in strategy game interfaces

Strategy games like Anno present players with enormous amounts of data simultaneously — population needs, production chains, trade balances, building queues, and diplomatic relations. When all of this information competes for attention, players struggle to identify what matters most at any given moment.

The existing UI treated most information with equal visual weight, making it difficult for players to quickly scan for the data they needed. Nested menus and inconsistent layout patterns added to the cognitive burden.

  • Too much information presented with equal visual weight
  • Nested menu structures that obscured critical data
  • Inconsistent layout patterns across different game systems
  • High cognitive load especially during time-sensitive gameplay moments

Process

Systematic analysis of information architecture

The process started with a thorough audit of the existing UI — documenting every screen, panel, and interaction pattern. Each interface element was categorized by frequency of use, criticality during gameplay, and the context in which players needed it.

Community feedback from forums and player reviews was analysed to identify the most common pain points. This player-sourced data informed the prioritization of which systems needed the most redesign attention.

UI Audit

Documented all existing screens and interaction patterns, mapping information density and visual hierarchy issues.

Player Research

Analysed community feedback, forum discussions, and player reviews to identify recurring UX pain points.

Systems Mapping

Created relationship diagrams between game systems to understand how information flows between different UI panels.

Solution

Layered information with clear visual hierarchy

The redesign proposal introduced a layered information architecture — a primary layer showing only the most critical real-time data, with secondary and tertiary layers accessible through progressive disclosure. This allowed players to scan essential information at a glance while diving deeper when needed.

Consistent design patterns were applied across all game systems: resource panels, building menus, and trade interfaces all followed the same visual language. Colour coding was standardized to indicate status (healthy, warning, critical) across every system.

The HUD was redesigned to be context-sensitive — showing relevant information based on what the player was currently doing rather than displaying everything at once.

Outcome

A framework for taming complexity

The case study produced a comprehensive redesign proposal with annotated mockups, interaction specifications, and a design rationale document. While this was a design exercise rather than a shipped product, the process yielded valuable insights into designing for information-dense applications.

The project reinforced that complex systems don't require complex interfaces — they require thoughtful information architecture that respects the player's attention and adapts to their current context.

  • Developed a layered information architecture reducing visible data by 40%
  • Created consistent design patterns applicable across all game systems
  • Designed context-sensitive HUD that adapts to player activity
  • Produced actionable redesign documentation with annotated mockups
